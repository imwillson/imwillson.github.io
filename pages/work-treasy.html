TravelEasy | github: imwillson/makeSchoolApp

Built an iOS application to find an optimal point for a driver to pickup a pedestrian who willing to walk toward the driver

Used: Swift, GoogleMapsGeometry Library, GoogleMaps API, Sketch (Prototyping)

Context

As a driver in NYC who also aggressively walks a lot, I always found myself using that background to move myself to a closer point whenever I was getting picked up by my Dad by car or vice versa. I wanted to find a solution to optimize this niche problem, instead of eyeballing it on a map. It found the point based on the "midpoint of time".

Made in a 8-week iOS Engineering Bootcamp called Make School.

Components

One ApplicationView to support the front-end

Google Maps API to find real-time information on distance/time between two points

Various helper classes and structs for storing data into JSON, coordinate conversion, distance calculation

Algorithm for finding the optimal point based on "midpoint of time". In the most basic explanation, if it took 10 minutes time for point A (driver) to drive to point B (passenger), the optimal point would be 5 minutes into the route

What I learned:

Never wait till the last minute to build the front end. Building the front helps me to visualize what is necessary to accomplish for the backend. Otherwise it is hard for me to iterate versions.

JSON format, pulling data from an API and reading the documentation for it,

Whenever there is redundancy in your code or a chance to make things more legible, modularize your code! Definitely helped me understand my code better since some functions turned out really verbose and it helped me test certain parts of my program more easily.

Ask for help! In my original algorithm, I was essentially using googleMaps API to binary search the most exactly halfway time, however, this was very resource intensive. I had to learn how to do some asynchronous programming because sometimes the function would take so long to process that the UI would not load. This was very innefficent and unnecessary. So my mentor helped me by constructing a new, but similar, way to find the optimal point.
